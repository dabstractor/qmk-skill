# `0.27.0` — keycode changes

<sub>2024-11-27 · commit `9f76541b29`</sub>

> One release in the QMK keycode migration catalog (`references/19-keycodes-changelog.md`). Apply the entries below only if your migration window `(from, to]` includes `0.27.0`.
>
> **Change-kind legend:** `deprecated OLD→NEW` → rename to **NEW** now (OLD still compiles via `quantum_keycodes_legacy.h`); `retired OLD` → replace with the **NEW** named where it was *deprecated* (OLD no longer compiles); `removed` → delete/substitute (no replacement); `added` → informational, not a migration requirement.

**Summary:** added 21 keycode(s): `QK_PERSISTENT_DEF_LAYER`, `QK_PERSISTENT_DEF_LAYER_MAX`, `QK_CONNECTION`, `QK_CONNECTION_MAX`, `QK_OUTPUT_AUTO`, `QK_OUTPUT_NEXT`, `QK_OUTPUT_PREV`, `QK_OUTPUT_NONE`, `QK_OUTPUT_USB`, `QK_OUTPUT_2P4GHZ` … (+11 more); added 6 macro(s): `IS_QK_PERSISTENT_DEF_LAYER(code)`, `IS_QK_CONNECTION(code)`, `IS_CONNECTION_KEYCODE(code)`, `IS_QUANTUM_KEYCODE(code)`, `CONNECTION_KEYCODE_RANGE`, `QUANTUM_KEYCODE_RANGE`; removed/renamed 3 keycode(s); added 2 macro(s): `PDF(layer)`, `QK_PERSISTENT_DEF_LAYER_GET_LAYER(kc)`; newly deprecated 1: `QK_OUTPUT_AUTO`→`OU_AUTO`
